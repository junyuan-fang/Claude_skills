# 微信通道回复尽量图文并茂

## 适用场景

用户通过 cc-connect 微信通道与 Claude 对话时,以及定时任务推送内容时。

## 规则

1. **需要配图的内容类型**:新闻速览、资讯总结、产品/公司介绍、行业动态、发布会回顾
2. **不需要配图的内容类型**:命令行操作回答、纯代码、调试输出、配置说明、原理解释
3. **图片获取方式**:
   - 从新闻原文页面抓配图(产品图/架构图/发布会截图)
   - 官方 logo / 产品图
   - 下载到本地后用 `cc-connect send --image <path>` 发送
4. **数量**:每条推送 1-3 张图为宜,流量与质量的平衡点

## 注意

- 部分新闻站点有反爬,抓不到图就只发文字,不要硬编造
- 创建/更新定时任务时,prompt 里要显式加上"抓 1-3 张配图一起发"的指令,daemon 才会执行
- 这是用户的持久偏好,应写入 feedback 记忆
