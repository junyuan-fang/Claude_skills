# 用户画像

## 偏好风格
- 偏好简洁汇总，不要中间态/工具调用步骤的状态刷屏，多步任务只在末尾一句总结
- 希望推送内容有深度展开（背景/技术/数据/业内对比），而不是只列关键词
- 喜欢用具体数字（金额、产能、参数量）和权威源链接支撑要点
- 推送内容里公司/产品业务背景必须带，但已熟的技术概念不要再解释
- 视频/竖屏内容偏好 YouTube Shorts 形态（1080×1920 / 9:16 / 60-90s）

## 沟通习惯
- 中文沟通，语气随意口语化（"你看着办"、"反正就类似这些"）
- 倾向迭代式反馈：先看效果再提改进，常要求"再跑一次看看"
- 会顺手抓事实错误（比如论文年份不对），期望立即修正并加防护规则

## 已知事实

## 注意事项
- 多步/后台任务执行中不要逐步播报进度，结束时给一条总结即可
- 新闻类 cron 红线：单图、不重试、sleep 30 之间隔、不发任何收尾 reply 状态消息
- 涉及"飞书"用语时确认实际通道，避免误导
- 解释概念时严格 gating：能搜到 Wikipedia 条目的不解释，只对新论文内部命名/小众 benchmark 1 句带过
- 公司业务背景（谁创立、估值、做什么）不属"已熟"，仍需带
- 引用 arXiv 论文为"今日新发"必须 14 天内：HuggingFace daily 优先，arXiv ID YYMM 粗筛 + abstract 页 `[Submitted on]` 精校；搜不到就放弃论文项，不包装老论文
