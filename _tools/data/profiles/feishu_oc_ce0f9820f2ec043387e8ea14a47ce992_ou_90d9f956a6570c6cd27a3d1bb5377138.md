# 用户画像

## 偏好风格
- 偏好静默执行：禁止任何中间态/收尾状态消息，只发实际内容
- 推送内容需"深度版"——每条带公司背景、技术细节、数字、业内对比中至少 2 项
- 反对关键词堆砌和并列符堆词，要求口语化 + 具体数字（x 倍 / y 万 / z 亿）
- 完成回执需一句话总结：路径 + 数量 + 状态，无需复述过程
- 配图严格 1 张，宁缺勿多，失败不重试（节流敏感）
- 推荐回执允许附带关键修正说明与趋势观察（如分类被精读修正、真空期说明、"零重合大丰收"、"WM 主题单日 6 篇"、"Alibaba AMAP 一日五发"、"Physics-informed WM 成流派"、"Cosmos 3 冠榜"、"VLA test-time 五部曲"、"WM as evaluator 罗盘"、"sim2real 第四派"、"WM 定义+罗盘时代"、"VLA foundation 数据军备"、"Sim 基础设施代际更替周"、"触觉 FM 六部曲"、"test-time 学习浪潮"、"RoboDojo 揭示 SOTA VLA 真机成功率仅 12.8%"等），不算"状态噪音"
- 回执可标注单 agent 分发覆盖不完整的情况（点名哪几篇必读未生成笔记）
- 回执可诚实标注数据源故障降级（如 7/13 arXiv API 全天 timeout 仅 HF trending 撑场），不视为借口噪音
- 必读笔记回执可附一句核心 benchmark 数字（如 WAM-TTT 真机 Progress 46.2% vs π0.5 14.8%）
- 问功能现状时期待结构化答复：现状（有无/开关状态）→ 关键机制要点 → 开启方法 → 主动提出补跑选项
